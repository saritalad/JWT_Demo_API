Data binding in Angular:
=============================

Data binding is fundamental concept in Angular that allows you to synchronize and maiplaute data between your component and view

TS code and your component template(HTML) .



Why do wee need Data binding ?


Whenever you want to develope any data driven web app, then as developer you need to keep the foucs on two important things that Data and UI(user interface) and its more imporatnt for you to find an efficent way to bind them(Data +UI) together.

Databinding in Angular classefied into 2 Cate:
---------------------------------------------------
		1. One Way Data binding(From component to View Template or From View to Component)
		2. Two Way data binding(From component to View and From view to component)



We have diff kind of Data biding apporaches in Angular:
	1. Interpolation (ONe-way data binding) : {{}}
	2. Property Data biding (One Way data binding) : [property]="expression"
	3. Event binding (One way data binding ) : (event)="expression"
	4. Two way data binding (ngModel)





One-Way : In the one-way binding data flows from one direction ,either from view to Component or From Componett to View:

Interpolation:{{}}:
=====================
Its allows us to include expression as part of any string literal, which we use in out HTML.

The angular evaluates the expression into a stirng and repalce it in the original string and update your view .

syntax:

================
As we already know every web app start with simeple first it take data then tranform the data into some format and finally , show the formated output ot the end user.
Getting the data is very simple , yyou careta a local variable itno complex type to hold the data or even you get the data from APIs.

The angular pipe(|) takes the raw data as input and then transform the raw data into some desigred format.

india=>(PIPE)=> INDIA 
For example:
Input data => Pipe= > final output


sanjay =>SANJAY =>Sanjay


Types of pipes:
==================
The Angular framework provide the pipes into two types:
	-Built-in Pipe
		-Paramtermeterzed
		-Chaining
	--Custom pipe


Angular pipe syntax:

 - expression : is the express which you want to tgranform
 | : is the pipe char
pipe operator : name of the pipe
pipeARgumnet: arguments to the pipe

expression|pipeOperator:pipeArgumnet


input(jay) => PIPE => JAY
